# timetrack.el

Emacs で日次の勤怠時間をプロジェクト・フェーズごとに管理するパッケージ。

## ファイル構成

```
~/.emacs.d/lisp/timetrack.el          パッケージ本体
~/works/notes/timetrack/
  projects.org                         プロジェクト/フェーズ定義
  timetrack_YYYYMMDD.org               日次記録 (自動生成)
```

## セットアップ

`init.el` に以下を追加する。

```elisp
(add-to-list 'load-path (expand-file-name "lisp" user-emacs-directory))
(require 'timetrack)
(global-set-key (kbd "C-c t") timetrack-map)
```

## キーバインド

`C-c t` がプレフィックス。

| キー      | コマンド                   |
|-----------|--------------------------|
| `C-c t t` | 今日のファイルを開く       |
| `C-c t a` | 工数エントリを追加         |
| `C-c t s` | 今日のサマリーを表示       |
| `C-c t S` | 日付指定でサマリーを表示   |
| `C-c t m` | プロジェクト定義を開く     |

## 使い方

### 1. プロジェクト定義を登録する

`C-c t m` で `projects.org` を開き、プロジェクト名とフェーズを登録する。

```org
#+TITLE: プロジェクト・フェーズ定義

* プロジェクト一覧

| プロジェクト名       | フェーズ | PJコード |
|---------------------|----------|----------|
| メニュー登録Ph1      | 設計     | PJ001    |
| メニュー登録Ph1      | 実装     | PJ001    |
| メニュー登録Ph1      | テスト   | PJ001    |
| デイリーミーティング | |         |
```

- フェーズが不要なプロジェクトはフェーズ列を空欄にする
- 同じプロジェクト名を複数行書くことでフェーズを追加できる

### 2. 工数を記録する

`C-c t a` で対話入力する。プロジェクト・フェーズは `projects.org` の内容で補完される。

```
プロジェクト: メニュー登録Ph1
フェーズ: 実装
タスク名: API開発
工数 (例: 1.5): 5.5
```

### 3. サマリーを確認する

`C-c t s` で当日分をプロジェクト・フェーズ別に集計表示する。

```
勤怠サマリー  2026-04-14
════════════════════════════════════════════════════════════

  ▌ デイリーミーティング
    (フェーズなし)            0.50 h

  ▌ メニュー登録Ph1
    設計                      1.25 h
    実装                      5.50 h
    ──────────────────────────────
    小計                      6.75 h

════════════════════════════════════════════════════════════
  合計                        7.25 h

[q] で閉じる
```

### 日次ファイルのフォーマット

`C-c t t` で作成される日次ファイルの構成。

```org
#+TITLE: 勤怠記録 2026-04-14
#+DATE: 2026-04-14

* 作業記録

| プロジェクト         | フェーズ | タスク名             | 工数 |
|---------------------|---------|---------------------|------|
| メニュー登録Ph1      | 設計    | 登録処理設計レビュー  | 1.25 |
| デイリーミーティング | | チーム朝会          | 0.5  |
| メニュー登録Ph1      | 実装    | API開発              | 5.5  |

-----

* タスク

-----

* メモ
```

## カスタマイズ

`init.el` で以下の変数を上書きできる。

```elisp
;; 格納ディレクトリの変更 (デフォルト: ~/works/notes/timetrack)
(setq timetrack-directory "~/path/to/your/dir")

;; 定義ファイル名の変更 (デフォルト: projects.org)
(setq timetrack-master-file "my-projects.org")
```
